IoT Fire Detection System

A project aimed at detecting fire in agricultural fields using a combination of smoke, temperature, and air quality indicators rather than relying on a single sensor.

Core Idea Combine multiple sensor readings to reduce false alarms caused by dust, fog, or momentary disturbances.
Sensors & Inputs Temperature, smoke, and AQI sensors feeding into a microcontroller that evaluates thresholds and patterns.
Alerts & Output When thresholds are crossed in a consistent pattern, alerts can be triggered for timely response.
Learning Outcome Practical understanding of sensor calibration, noise, and the gap between ideal and real-world data.

Algorithm Visualizer

A C++-based project using Raylib to visually show how algorithms behave over time instead of just printing outputs.

Sorting Algorithms Visual bars move and swap as algorithms like bubble, selection, or quicksort run step by step.
Pathfinding Grids and nodes show how traversal algorithms explore paths and backtrack.
Backtracking Visual traces of decisions and rollbacks that are usually hidden in console programs.
Understanding Time Connecting time complexity to how long the visualizations actually take when input size grows.

Tourist Reservation System

A C++ OOP console project that simulates a reservation system for tourists, focusing on structure and reliability.

Core Features Booking, searching, and cancelling reservations using clean menu flows.
Object-Oriented Design Classes for customers, bookings, and destinations to keep logic organized.
File Handling Data persistence using file operations so that information is not lost between runs.
Error Handling Checking inputs, handling invalid choices, and preventing crashes on bad data.

Postman Library API v2

A REST-style project modeled as a library system, tested thoroughly using Postman collections.

Endpoints Standard CRUD operations for books, users, and borrow/return flows.
Postman Collections Organized requests with environments to quickly verify behaviour and regressions.
Error Scenarios Testing invalid data, missing fields, and edge cases instead of only happy paths.
Takeaways Clear understanding of request/response cycles, status codes, and API debugging.
